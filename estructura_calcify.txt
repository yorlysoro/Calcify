# Calcify — Reporte de Estado del Proyecto

## 1. Resumen General

Calcify es una aplicación de escritorio para gestión de inventario y
divisas (currency/inventory management) construida con Flask 3 +
SQLAlchemy 2.0 bajo Clean Architecture (4 capas estrictas).

- **Versión actual:** 0.7.0 (en desarrollo activo)
- **Python:** 3.13+
- **Framework web:** Flask 3.1.3
- **ORM:** SQLAlchemy 2.0.49
- **Base de datos:** SQLite (con migraciones automáticas vía Alembic 1.18.4)
- **Tests:** Pytest 9.0.3 (108 tests)
- **Cobertura:** 80% (892/1110 statements)

## 3. Capas de Arquitectura (Clean Architecture)

Reglas estrictas de dependencia (una vía, de afuera hacia adentro):

| Capa | Depende de | Imports prohibidos |
|------|-----------|-------------------|
| `domain/` | Python stdlib | Flask, SQLAlchemy, cualquier framework |
| `use_cases/` | Entidades domain + interfaces abstractas | ORM models, Flask |
| `infrastructure/` | SQLAlchemy, entidades domain, interfaces | Flask globals (request, g, session) |
| `presentation/` | Flask, entidades domain, repos concretos | ORM models directamente |

## 4. Entidades del Dominio (domain/models.py)

### Currency
- code: str (PK, ISO 4217, ej. "USD")
- name: str
- symbol: str
- is_main: bool

### ExchangeRate (legacy)
- base_currency_code: str
- target_currency_code: str
- rate: Decimal
- date: datetime

### Product
- id: UUID (PK)
- name: str
- cost_price: Decimal
- cost_currency_code: str (FK → currencies.code)
- margin_percentage: Decimal
- category: str = "Uncategorized"
- stock_quantity: int = 0
- calculate_sale_price() → Decimal (cost_price * (1 + margin/100))
- get_sale_price_in_currency(...) → Decimal (convierte a otra moneda)

### CurrencyRate
- id: UUID (PK)
- currency_code: str
- rate: Decimal
- inverse_rate: Decimal = 0.0
- created_at: datetime (timezone-aware)

### Transaction
- id: UUID (PK)
- product_id: UUID (FK → products.id)
- transaction_type: str ("IN" | "OUT")
- quantity: int
- unit_price: Decimal
- currency_code: str
- created_at: datetime (timezone-aware)
- comment: str = ""

## 5. Servicios del Dominio (domain/services/currency_converter.py)

### CurrencyConverter.convert(amount, source_inverse, target_inverse) → Decimal
- Fórmula: (amount * target_inverse) / source_inverse
- Quantize a 4 decimales con ROUND_HALF_UP
- Lanza InvalidExchangeRateError si algún inverse_rate es 0

## 8. Autenticación (presentation/api/auth.py)

- POST /login — espera {"pin": "..."}, verifica contra hash en DB
- POST /logout — limpia sesión
- @login_required — decorator que retorna 401 (API) o redirect (web)
- Session-based auth (no JWT)
- Session fixation prevention: session.clear() antes de login exitoso
- CSRF no implementado (app de escritorio single-user)

## 10. Migraciones Automáticas (infrastructure/database/auto_migrate.py)

bootstrap_migrations(db_url, metadata) ejecuta en cada boot:

1. Migra artefactos legacy (alembic.ini, migrations/, schema_trace.sql)
   del project root al AppData dir
2. Inicializa Alembic (alembic.ini + migrations/) si no existen
3. Parchea env.py con sys.path, target_metadata, render_as_batch
4. Detecta estado de la DB: fresh, tables_exist_no_tracking, o normal
5. ALTER TABLE repair para columnas faltantes
6. Detecta diferencias de schema (tipos, columnas faltantes/sobrantes)
7. Ejecuta comandos Alembic según el estado:
   - Fresh DB → revision(autogenerate=True) + upgrade
   - Sin tracking → revision(no autogenerate) + stamp
   - Diferencias no-aditivas → revision(autogenerate=True) + upgrade
   - Normal → upgrade("head")
8. Maneja CommandError: "No changes" (log), "Can't locate revision"
   (DROP alembic_version + nueva baseline + stamp), otros (re-raise)
9. Genera schema_trace.sql (DDL offline dump)
